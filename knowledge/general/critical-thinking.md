# Critical Thinking & Decision Making Framework

## Overview
Critical thinking is the disciplined process of analyzing, evaluating, and synthesizing information to form sound judgments. This framework provides structured approaches for better decisions in complex, uncertain environments.

## Core Mental Models

### 1. First Principles Thinking
```
Break down to fundamentals:
1. Identify assumptions
2. Strip to fundamental truths (physics, math, human nature)
3. Rebuild from ground up
4. Test each reconstruction

Example: "Batteries are too expensive"
- Assumption: Batteries cost $X/kWh
- Fundamental: Materials (Li, Co, Ni) cost $Y/kg
- Rebuild: New chemistries, manufacturing, recycling
- Result: Path to $Z/kWh
```

### 2. Inversion (Jacob Mikanowski / Charlie Munger)
```
Instead of "How do I succeed?" ask "How do I fail?"
1. Identify goal
2. List all ways to fail
3. Avoid those failure modes
4. What remains is path to success

Example: Want successful marriage
→ How to fail: Ignore partner, never communicate, prioritize self, hide finances
→ Avoid these → Path to success
```

### 3. Second-Order Thinking
```
First-order: Immediate consequence
Second-order: Consequence of consequence
Third-order: Systemic/long-term effects

Always ask: "And then what?"

Example: Rent control
1st: Rents stay low
2nd: Landlords defer maintenance, convert to condos
3rd: Housing supply shrinks, quality drops, black market emerges
```

### 4. Probabilistic Thinking
```
Replace "Will X happen?" with "What's P(X)?"
- Assign probabilities to outcomes
- Update with new evidence (Bayesian)
- Make decisions based on expected value
- Track calibration (were my 70% predictions right 70% of time?)

Tools:
- Fermi estimation (order-of-magnitude)
- Monte Carlo simulation (complex)
- Prediction markets (aggregated wisdom)
```

### 5. Systems Thinking
```
Elements → Interconnections → Function → Emergence

Look for:
- Feedback loops (reinforcing R, balancing B)
- Delays (cause → effect lag)
- Stocks & flows (accumulations vs rates)
- Leverage points (small change → big effect)

Iceberg model:
Events (what happened)
→ Patterns (trends)
→ Structures (systems causing patterns)
→ Mental models (beliefs driving structures)
```

## Decision-Making Frameworks

### WRAP Process (Heath Brothers)
```
W - Widen your options
  - Avoid narrow framing ("whether or not")
  - Multitrack (test multiple options)
  - Find someone who solved it
  - Analogize (other domains)

R - Reality-test your assumptions
  - Consider opposite
  - Zoom out (base rates), zoom in (details)
  - Ooch (small experiments)
  - Ask disconfirming questions

A - Attain distance before deciding
  - Shift perspective (10/10/10 rule)
  - Set tripwires (automatic review triggers)
  - Honor core priorities

P - Prepare to be wrong
  - Bookend (premortem + preparade)
  - Set tripwires
  - Build in safety margins
```

### OODA Loop (Boyd)
```
Observe → Orient → Decide → Act → (loop)

Speed matters: Faster cycles beat better analysis
- Observe: Gather data, filter noise
- Orient: Update mental models, challenge biases
- Decide: Select hypothesis to test
- Act: Execute, gather feedback
- Repeat faster than environment changes
```

### Cynefin Framework (Snowden)
```
Domain              | Approach
Clear (Ordered)     | Sense → Categorize → Respond (Best practice)
Complicated         | Sense → Analyze → Respond (Good practice)
Complex             | Probe → Sense → Respond (Emergent practice)
Chaotic             | Act → Sense → Respond (Novel practice)
Disorder            | Identify domain first
```

### Decision Matrix (Weighted Scoring)
```
Criteria (weight)    | Option A | Option B | Option C
Cost (0.30)          |    7     |    5     |    8
Time (0.25)          |    5     |    8     |    6
Quality (0.25)       |    8     |    7     |    5
Risk (0.20)          |    6     |    6     |    7

```

## Cognitive Bias Mitigation

### Common Biases & Counters
| Bias | Description | Counter |
|------|-------------|---------|
| Confirmation | Seek confirming evidence | Actively seek disconfirming evidence |
| Availability | Judge by ease of recall | Seek base rates, statistics |
| Anchoring | Overweight first info | Set own anchor first, seek range |
| Overconfidence | Overestimate accuracy | Pre-mortem, calibration training |
| Sunk cost | Honor past investment | Ignore sunk costs, decide forward |
| Status quo | Prefer current state | Reframe as active choice |
| Framing | React to presentation | Reframe both ways |
| Halo effect | One trait colors all | Evaluate dimensions separately |
| Recency | Overweight recent | Weight by relevance, not time |
| Survivorship | Study winners only | Study failures too |

### Bias Mitigation Checklist
```
Before major decision:
□ List assumptions
□ Identify base rates
□ Seek disconfirming evidence
□ Conduct pre-mortem
□ Sleep on it (24hr rule)
□ Get outside view
□ Check for conflict of interest
□ Assign devil's advocate
```

## Problem-Solving Process

### Define → Explore → Decide → Execute → Learn

#### 1. Define (Problem Framing)
```
- Write problem statement (one sentence)
- Distinguish symptoms vs root cause (5 Whys)
- Define success criteria (SMART)
- Identify constraints & stakeholders
- Determine decision type: Type 1 (irreversible) vs Type 2 (reversible)
```

#### 2. Explore (Generate Options)
```
- Diverge before converge
- Quantity → Quality
- Techniques:
  * Brainwriting (silent, written)
  * SCAMPER (Substitute, Combine, Adapt, Modify, Put to other use, Eliminate, Reverse)
  * Six Thinking Hats
  * Worst possible idea (then invert)
  * Analogical thinking (other domains)
- No criticism during generation
```

#### 3. Decide (Evaluate & Choose)
```
- Apply decision criteria matrix
- Consider reversible vs irreversible
- Use appropriate framework (WRAP, OODA, Cynefin)
- Document rationale (decision log)
- Set review triggers (tripwires)
```

#### 4. Execute (Implementation)
```
- Break into next actions (GTD)
- Assign owners, deadlines
- Identify risks & mitigations
- Build feedback loops
- Communicate clearly (who, what, when, why)
```

#### 5. Learn (Review & Iterate)
```
After-action review:
- What was intended?
- What happened?
- Why the difference?
- What to sustain/improve?
- Update mental models
```

## Root Cause Analysis

### 5 Whys
```
Problem: Machine stopped
1. Why? Fuse blew → 2. Why? Bearing seized → 3. Why? No lubrication → 4. Why? Pump failed → 5. Why? No filter → metal shavings
→ Root cause: No filter on lubrication pump
→ Countermeasure: Install filter, add maintenance check
```

### Fishbone (Ishikawa) Diagram
```
Categories: 6Ms
- Man (people)
- Machine (equipment)
- Material
- Method (process)
- Measurement
- Mother Nature (environment)

Problem at head, causes on bones, sub-causes on smaller bones
```

### Fault Tree Analysis
```
Top event (failure) → AND/OR gates → Basic events
Quantitative: Assign probabilities, calculate top event probability
Used in: Safety engineering, reliability
```

- **Evening**: Review decisions, note surprises, update beliefs

### Calibration Training
```
Weekly: Make 5-10 probabilistic predictions
- "Project X ships by Friday: 70%"
- "Stock Y up 5% this month: 40%"
- "Candidate Z wins: 60%"

Track: Actual outcomes vs predictions
Goal: Calibration curve approaches diagonal
Tools: Metaculus, Good Judgment Open, personal spreadsheet
```

## Group Decision Making

### Avoiding Groupthink
- Assign devil's advocate (rotate)
- Anonymous input first (brainwriting)
- Leader speaks last
- Encourage dissent (reward it)
- Outside experts
- Red team / Blue team

### Consensus vs Consent
- **Consensus**: Everyone agrees (slow, fragile)
- **Consent**: No reasoned objection (faster, robust)
- **Advice process**: Decider consults affected/experts, then decides
- **Benevolent dictator**: One decides, owns outcome

### Decision Rights (RAPID)
```
R - Recommend (proposes)
A - Agree (must approve)
P - Perform (executes)
I - Input (consulted)
D - Decide (single owner)
```

```

Convert: Each failure → preventive action
Assign: Owner + deadline for each prevention
```

### Tripwire Examples

- "If budget exceeds 120% → auto-review"
- "If key person leaves → activate succession plan"
- "If competitor launches X → reassess roadmap"

## Advanced Techniques

### Red Teaming
- Independent group attacks plan
- Adversarial perspective
- Find blind spots, weaknesses
- Used in: Military, intelligence, cybersecurity, strategy

### Scenario Planning
```
1. Identify key uncertainties (2-3 critical variables)
2. Create 4 scenarios (2×2 matrix)
3. Flesh out narratives
4. Identify early indicators
5. Develop robust strategies (work across scenarios)
6. Monitor signposts
```

### Wardley Mapping
```
Visualize value chain:
User needs → Components → Evolution (Genesis → Custom → Product → Commodity)
Identify: Strategic positions, duplication, outsourcing opportunities
```

### Monte Carlo Simulation
```
For complex decisions with many uncertain variables:
1. Define probability distributions for inputs
2. Run 10,000+ simulations
3. Analyze output distribution
4. Decision: P(success) > threshold
Tools: @Risk, Crystal Ball, Python (PyMC3), Excel add-ins
```
