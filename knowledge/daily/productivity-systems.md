# Personal Productivity Systems

## Overview
Productivity isn't about doing more—it's about doing what matters effectively. A good system reduces cognitive load, captures everything, and enables focus on high-impact work.

## Core Principles

### Capture Everything
- **Trusted system**: Brain is for thinking, not remembering
- **Inboxes**: Physical + digital, processed regularly
- **Ubiquitous capture**: Phone, watch, paper, voice

### Clarify & Organize
- **Next action**: Specific, physical, visible step
- **Projects**: Any outcome requiring >1 action
- **Contexts**: Where/tool/person needed (@computer, @phone, @errands, @home)
- **Waiting for**: Delegated items with follow-up dates

### Review Frequently
- **Daily**: Calendar, inboxes, 3 MITs (Most Important Tasks)
- **Weekly**: Projects, waiting for, someday/maybe, calendar review
- **Monthly/Quarterly**: Goals, roles, systems audit

### Engage Intentionally
- **Energy management** > time management
- **Deep work** blocks for cognitive tasks
- **Shallow work** batched (email, admin, meetings)
- **Single-tasking** with clear completion criteria

## Popular Frameworks

### GTD (Getting Things Done) - David Allen
```
5 Steps:
1. CAPTURE → Inboxes
2. CLARIFY → Process inboxes (actionable? project? reference? trash?)
3. ORGANIZE → Lists (Next Actions, Projects, Waiting For, Someday/Maybe, Calendar)
4. REFLECT → Weekly Review
5. ENGAGE → Do (context, time, energy, priority)

Key Lists:
- Next Actions (by context)
- Projects (active)
- Waiting For (with dates)
- Someday/Maybe (incubator)
- Calendar (hard landscape only)
- Reference (file system)
```

### Time Blocking / Calendar Blocking
```
Principles:
- Every minute assigned a purpose
- Include buffers (transition, overflow)
- Protect deep work blocks (90-120 min)
- Theme days (Admin Monday, Deep Work Tue/Thu, Meetings Wed)

Template:
6:00  Morning routine
7:00  Deep Work Block 1
9:00  Break / shallow
9:30  Deep Work Block 2
11:30 Email / comms
12:00 Lunch / walk
13:00 Meetings / collab
15:00 Admin / shallow
16:00 Buffer / catch-up
17:00 Shutdown ritual
```

### PARA (Projects, Areas, Resources, Archives) - Tiago Forte
```
Projects: Active, deadline-driven (publish article, launch feature)
Areas: Ongoing responsibilities (health, finances, career)
Resources: Reference material (research, templates, articles)
Archives: Completed/inactive items

Digital filing: Same structure everywhere (files, notes, bookmarks)
```

### Eisenhower Matrix
```
                    URGENT              NOT URGENT
IMPORTANT      DO now (crises)      SCHEDULE (strategic)
NOT IMPORTANT  DELEGATE (interrupt) DELETE (distractions)
```

### Ivy Lee Method (Daily)
```
End of day:
1. Write 6 most important tasks for tomorrow
2. Prioritize 1-6
3. Next day: Start with #1, finish before #2
4. Move unfinished to next day's list
5. Repeat daily
```

### Eat the Frog (Brian Tracy)
- Do hardest, most important task FIRST
- "If you eat a frog first thing, nothing worse happens all day"
- Identify frog the night before

## Email Management

### Inbox Zero Philosophy
- Inbox = temporary holding, not storage
- Process to zero regularly (not constantly)
- Actions: Delete, Delegate, Defer, Do (<2 min), File

### Strategies
```
1. Batch processing: 2-3x/day (morning, lunch, end)
2. Turn off notifications
3. Unsubscribe ruthlessly
4. Filters/rules for newsletters, receipts, CC
5. Templates/canned responses for common replies
6. Two-minute rule: If <2 min, do now
7. Archive aggressively (search > folders)
```

### Folder Structure (Minimal)
```
Inbox (processing only)
Action Required (deferred)
Waiting For (delegated)
Reference (searchable archive)
Projects (active project support)
```

## Calendar Management

### Principles
- **Single calendar** (personal + work combined view)
- **Default to busy** for focus time
- **Buffer time** between meetings (15 min)
- **Meeting norms**: Agenda required, start/end on time, decline if not needed
- **Theme days** for context switching reduction

### Time Blocking Types
| Block Type | Duration | Purpose |
|------------|----------|---------|
| Deep Work | 90-120 min | Cognitive heavy lifting |
| Shallow Work | 30-60 min | Email, admin, Slack |
| Meetings | 25/50 min | Collaboration (25/50 rule) |
| Buffer | 10-15 min | Transition, overflow |
| Recovery | 15-30 min | Walk, stretch, meditate |
| Personal | Variable | Exercise, family, errands |

## Focus & Deep Work

### Pomodoro Technique
```
1. Pick task
2. Set timer (25 min)
3. Work until timer
4. Short break (5 min)
5. Every 4 cycles: long break (15-30 min)

Variations:
- 52/17 (DeskTime research)
- 90/20 (ultradian rhythm)
- Flowmodoro: Work until flow breaks, break proportional
```

### Distraction Management
```
Digital:
- Phone: Grayscale, focus modes, out of sight
- Browser: Site blockers (Freedom, Cold Turkey), tab limiters
- Notifications: Off by default, VIP only
- Separate browser profiles (work/personal)

Physical:
- Clean desk (only current project)
- Noise: Noise-cancelling, white/brown noise, focus music
- Lighting: Bright cool for focus, warm for evening
- Door sign / status indicator
```

### Attention Residue Prevention
- Complete task or reach clear stopping point before switching
- Write "next action" note when interrupted
- Context-switching cost: 15-23 min to regain deep focus

## Energy Management

### Chronotypes
| Type | Peak Alertness | Best Deep Work | Population |
|------|----------------|----------------|------------|
| Lion (Early) | 6-10 AM | Morning | 15% |
| Bear (Standard) | 10 AM-2 PM | Late morning | 55% |
| Wolf (Late) | 4 PM-12 AM | Evening | 15% |
| Dolphin (Irregular) | Varies | When able | 10% |

### Ultradian Rhythms
- 90-min focus cycles followed by 20-min rest
- Signs of needing break: yawning, fidgeting, distraction, errors
- Honor the dip, don't push through with caffeine

### Sleep Hygiene (Foundation)
- 7-9 hours, consistent schedule
- Cool (65-68°F), dark, quiet
- No caffeine after 2 PM, no alcohol 3 hrs before
- Morning light (10 min within 30 min waking)
- Wind-down routine (no screens 60 min before)

## Habit Formation

### Habit Loop (Duhigg)
```
Cue → Craving → Response → Reward

Design:
- Cue: Obvious (time, location, preceding event, emotional state, people)
- Craving: Attractive (temptation bundling, social norm)
- Response: Easy (2-min rule, environment design, automation)
- Reward: Satisfying (immediate feedback, habit tracking, identity)
```

### Atomic Habits (Clear) - 4 Laws
1. **Make it Obvious**: Implementation intention ("I will [BEHAVIOR] at [TIME] in [LOCATION]"), habit stacking, environment design
2. **Make it Attractive**: Temptation bundling, join culture where desired behavior is normal
3. **Make it Easy**: Reduce friction, 2-minute rule, automate, prime environment
4. **Make it Satisfying**: Immediate reward, habit tracker, accountability partner, identity-based ("I'm a writer" vs "I want to write")

### Habit Stacking Formula
```
After [CURRENT HABIT], I will [NEW HABIT].

Examples:
- After I pour coffee, I will write 3 gratitudes
- After I brush teeth, I will floss one tooth
- After I close laptop, I will write tomorrow's MITs
```

## Weekly Review Template (60-90 min)
```
1. GET CLEAR
   ☐ Clear physical inbox
   ☐ Clear digital inboxes (email, messages, notes)
   ☐ Capture loose thoughts/ideas

2. GET CURRENT
   ☐ Review calendar (past week, next 2 weeks)
   ☐ Review project lists (complete? next actions?)
   ☐ Review waiting for list (follow up?)
   ☐ Review someday/maybe (activate? delete?)

3. GET CREATIVE
   ☐ Review goals/OKRs (on track?)
   ☐ Brainstorm new projects/ideas
   ☐ Plan next week (3 MITs per day, time blocks)
   ☐ Identify bottlenecks/obstacles

4. SYSTEM MAINTENANCE
   ☐ Backup/cleanup digital files
   ☐ Update tools/apps
   ☐ Refine system friction points
```

## Goal Setting

### OKRs (Objectives & Key Results)
```
Objective: Qualitative, ambitious, time-bound
Key Results: Quantitative, measurable, 3-5 per objective

Example:
O: Become a recognized expert in WebRTC
KR1: Publish 12 technical articles (1/month)
KR2: Speak at 3 conferences
KR3: Contribute to 2 open-source WebRTC projects
KR4: Grow newsletter to 5,000 subscribers
```

### SMART Goals
- **S**pecific, **M**easurable, **A**chievable, **R**elevant, **T**ime-bound

### Yearly → Quarterly → Monthly → Weekly → Daily
- Annual themes/directions
- Quarterly objectives (OKRs)
- Monthly focus areas
- Weekly priorities (3-5)
- Daily MITs (1-3)

## Decision Making

### Two-Way Door vs One-Way Door (Bezos)
- **Type 1 (One-way)**: Irreversible, high stakes → deliberate, gather data
- **Type 2 (Two-way)**: Reversible, low stakes → decide fast, iterate

### Regret Minimization Framework
- Project to age 80: Which choice minimizes regret?

### 10-10-10 Rule (Suzy Welch)
- How will I feel in 10 minutes? 10 months? 10 years?

### Reversibility Test
- "If this fails, can I undo it easily?" → If yes, just do it

## System Maintenance

### Quarterly Audit (2-3 hours)
- [ ] Review all projects (archive completed)
- [ ] Audit areas of responsibility
- [ ] Clean reference files
- [ ] Evaluate tools (consolidate, upgrade, remove)
- [ ] Assess habits (keep, modify, drop)
- [ ] Calendar audit (recurring meetings valuable?)
- [ ] Goal progress check
- [ ] System friction log review

### Friction Logging
- Throughout day: Note "This took too long" or "I forgot to..."
- Weekly: Review, identify pattern, design fix
- Example: "Forgot to log time" → Auto-timer + daily reminder
