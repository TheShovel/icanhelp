# Note-Taking Methods

Systems for capturing, organizing, and retrieving information from lectures, books, and meetings.

## Cornell Method
- **Divide page**: 2.5-inch left column (Cue), 6-inch right column (Notes), 2-inch bottom (Summary)
- **Notes column**: Capture main ideas during lecture — shorthand, abbreviations, bullet points
- **Cue column**: After class, write questions, keywords, or prompts that relate to notes — cover notes and recall from cues
- **Summary**: After review, write 1-3 sentences summarizing the page's content
- Best for: Lectures, textbooks, meetings — structured review, self-testing built in

## Zettelkasten (Slip Box)
- **Atomic notes**: One idea per note — concise, self-contained, written in your own words
- **Links**: Each note connects to others — `[[related-idea]]` or unique IDs (timestamps)
- **Index**: Entry points for topics — map of content notes that link to key atomic notes
- **Connection > collection**: Value is in links between ideas, not number of notes
- **Digital tools**: Obsidian, Logseq, Roam Research — bidirectional links, graph view, backlinks, full-text search
- Workflow: Fleeting notes (quick capture) → Literature notes (source processing) → Permanent notes (refined, linked)

## Outline Method
- **Hierarchical structure**: Main topic → Subtopic → Detail (I. / A. / 1. / a.)
- Indent sub-points under main points — shows relationships and hierarchy at a glance
- **Speed**: Fast to write during lectures — use abbreviations, symbols (&, →, ≈, ≠)
- Best for: Well-structured lectures, textbooks with clear headings — not for complex interconnected topics

## Mind Mapping
- **Central idea** in middle → branches radiate out for each subtopic → sub-branches for details
- Use keywords (not full sentences), colors, images — stimulates both brain hemispheres
- **Organic structure**: Add branches wherever they connect — non-linear, shows relationships
- **Review**: Quick visual scan shows the whole topic — easier than reading linear notes
- Tools: FreeMind, XMind, mindmaps (physical), Miro, Draw.io
- Best for: Brainstorming, project planning, creative topics, connecting ideas

## Bullet Journal (BuJo)
- **Rapid logging**: Short bullets — tasks (·), events (○), notes (−) — migrated/checked as done
- **Collections**: Index (table of contents), Future Log (12-month view), Monthly Log (calendar + tasks), Daily Log
- **Migration**: Review monthly — migrate unfinished tasks to next month — intentionality, only keep what matters
- **Signifiers**: `*` priority, `!` inspiration, `?` need research, `<` scheduled, `>` migrated
- Best for: Personal organization, habit tracking, combining calendar + tasks + notes in one system

## Charting Method
- **Columns**: One column per category/variable — row per item or time period
- **Rows**: Observations or data points — easy to compare across categories
- Best for: Comparing pros/cons, tracking changes over time, scientific observations, history timelines

## Sentence Method
- **Every new thought** on a separate line — numbered for reference
- **No structure**: Fast but hard to review — relationships between ideas not shown
- Best for: Last resort when lecture is too fast for other methods — re-write into better format later

## Digital Note Apps Comparison
- **Obsidian**: Markdown files locally stored — graph view, plugins, community — best for Zettelkasten
- **Notion**: All-in-one — databases, wiki, kanban, calendar — collaborative, but slow, not offline-first
- **Evernote**: Web clipper, PDF search — aging, limited markdown support
- **OneNote**: Free-form canvas, draws with stylus — good for handwritten + typed mixed notes
- **Roam Research**: Block-level references, daily notes, bidirectional linking — outliner format
- **Logseq**: Open-source, local-first, outliner + markdown — block references, journal-based
- **Apple Notes**: Simple, syncs with iCloud, quick capture — limited organization
- **Google Keep**: Sticky notes, reminders, checklists — too simple for deep notes, good for quick capture
