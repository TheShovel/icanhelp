# Resume Writing & Job Search Strategy

## Overview
A resume is a marketing document, not a historical record. Its sole purpose: get you an interview. Every element should serve that goal.

## Resume Structure

### Standard Sections (In Order)
1. **Header** - Name, contact, links
2. **Professional Summary** - 2-3 lines, tailored
3. **Technical Skills** - Categorized, current
4. **Experience** - Reverse chronological
5. **Projects** - 2-3 relevant (optional for senior)
6. **Education** - Degree, school, graduation year
7. **Certifications/Awards** - If relevant

### Header Template
```
FIRST LAST
City, State | Phone | Email | LinkedIn | GitHub | Portfolio
```

### Professional Summary (2-3 lines)
**Formula**: [Years] [Title] specializing in [Core Skills]. Proven track record of [Key Achievement with Metric]. Seeking to leverage [Strength] at [Target Company Type].

**Example**:
> Senior Software Engineer with 8+ years building scalable distributed systems. Specializing in Go, Kubernetes, and event-driven architectures. Reduced infrastructure costs 40% by migrating monolith to microservices on GKE. Seeking to apply platform engineering expertise at a Series B+ startup.

## Experience Bullets: The STAR Method

### Formula
**Action Verb** + **Context/Challenge** + **Action Taken** + **Quantified Result**

### Strong vs Weak Bullets

| Weak | Strong |
|------|--------|
| Responsible for backend development | Built REST APIs serving 10M+ requests/day with 99.99% uptime |
| Worked on database optimization | Reduced query latency from 2.3s to 45ms by adding composite indexes and partitioning |
| Managed CI/CD pipelines | Cut deployment time from 45min to 8min by migrating Jenkins to GitHub Actions with parallel jobs |
| Wrote unit tests | Increased code coverage from 34% to 87% by implementing pytest + CI gate |
| Led team of 5 engineers | Mentored 5 engineers; 3 promoted within 18 months; established code review standards |

### Action Verbs by Category
| Category | Verbs |
|----------|-------|
| **Technical** | Architected, Engineered, Optimized, Refactored, Implemented, Designed, Deployed, Automated |
| **Leadership** | Led, Directed, Mentored, Coached, Spearheaded, Championed, Orchestrated |
| **Impact** | Increased, Reduced, Accelerated, Delivered, Achieved, Generated, Saved |
| **Collaboration** | Collaborated, Partnered, Facilitated, Coordinated, Aligned |

## Tailoring for Each Application

### Keyword Matching Process
1. **Extract keywords** from job description (skills, tools, methodologies)
2. **Audit your resume** - which keywords are missing?
3. **Add missing keywords** honestly where you have experience
3. **Reorder skills** - most relevant first
4. **Adjust summary** - mirror their language

### ATS Optimization
- Use standard section headers
- Avoid columns, tables, graphics
- Save as PDF (unless .docx requested)
- Include both acronym and full term: "Kubernetes (K8s)"
- Use standard job titles

## Experience Section Details

### What to Include Per Role
```
Company Name | Location | Month/Year – Month/Year
Role Title

• [Strongest bullet with biggest metric]
• [Second strongest with different skill]
• [Third bullet showing breadth or leadership]
• [Optional 4th if highly relevant]
```

### Handling Common Situations

| Situation | Strategy |
|-----------|----------|
| **Short tenure (<1 year)** | Include if achievement-rich; group short contracts |
| **Employment gap** | Brief note: "Sabbatical," "Caregiver leave," "Upskilling" |
| **Career pivot** | Highlight transferable skills; add relevant projects |
| **Multiple roles same company** | List company once, roles as sub-bullets |
| **Contract/Freelance** | Label clearly; group under "Contract Roles" |
| **Startup acquired** | Note: "Acquired by [Company]" |
| **Confidential work** | Describe generically: "Fintech platform processing $XB annually" |

## Projects Section (When to Include)
- **Junior/Mid**: 2-3 projects showing depth
- **Career changers**: Critical - proves skills
- **Senior**: Only if flagship/open source
- **Format**: Name | Tech Stack | 1-line impact + link

```
E-Commerce Platform (Go, PostgreSQL, Kubernetes, gRPC)
Designed and built high-throughput order processing system handling 50K orders/day.
Implemented idempotent payment processing with exactly-once semantics.
GitHub: github.com/user/ecommerce | Live: demo.example.com
```

## Cover Letter Template

### Structure (3-4 paragraphs, <300 words)
```
Dear [Hiring Manager Name],

[Hook: Specific reason for interest in THIS company/role]

[Body: 2-3 sentences connecting your top 2-3 achievements to their needs.
Use their keywords. "Your job description mentions X - at Company Y, I..."]

[Closing: Enthusiasm + call to action]
I'd welcome the chance to discuss how my experience with [Key Skill] 
can help [Company] achieve [Their Goal]. Thank you for your consideration.

Best regards,
[Your Name]
```

### Customization Checklist
- [ ] Addressed to specific person (LinkedIn search)
- [ ] Mentions company's specific product/mission
- [ ] References 1-2 specific job requirements
- [ ] Shows you researched them (recent news, blog, tech blog)
- [ ] No generic "I'm a hard worker" fluff

## Job Search Strategy

### Application Funnel Targets
| Stage | Target | If Below |
|-------|--------|----------|
| Applications → Screen | 15-20% | Improve resume tailoring |
| Screen → Technical | 40-50% | Practice phone screens |
| Technical → Onsite | 40-50% | Practice coding/design |
| Onsite → Offer | 50-60% | Negotiate, prep behavioral |

### Application Volume
- **Active search**: 15-25 quality applications/week
- **Passive**: 3-5/week
- **Quality > Quantity**: 10 tailored > 100 spray-and-pray

### Application Tracking
Use a spreadsheet or tool (Notion, Huntr, Teal):
| Company | Role | Date Applied | Status | Contact | Next Step | Notes |
|---------|------|--------------|--------|---------|-----------|-------|

## Networking for Referrals

### LinkedIn Outreach Template
```
Hi [Name],

I'm a [Your Title] exploring opportunities at [Company] and noticed your work on [Specific Project/Team]. 

I've been following [Company]'s work in [Domain] - particularly [Specific Detail from Blog/News]. 

Would you be open to a brief 10-min chat about your experience on the [Team] team? No pressure at all - just looking to learn.

Thanks,
[Your Name]
```

### Referral Request (After Connection)
```
Thanks for the chat, [Name]! 

I'm applying for the [Role] position (req #12345). Would you be comfortable referring me? 

I've attached my tailored resume - happy to send a 3-bullet summary you can forward if that's easier.

Either way, really appreciated the conversation!
```

## Interview Preparation

### Technical Prep by Level
| Level | Focus Areas |
|-------|-------------|
| **Junior** | Data structures, algorithms, OOP, debugging, eagerness |
| **Mid** | System design (basics), trade-offs, testing, mentoring |
| **Senior** | System design (deep), architecture decisions, trade-offs, leadership |
| **Staff/Principal** | Strategy, org influence, cross-team, technical vision |

### System Design Prep
1. **Clarify requirements** (5 min) - scale, features, non-functional
2. **High-level design** (10 min) - API, data model, components
3. **Deep dive** (15 min) - 2-3 critical components
4. **Bottlenecks & scaling** (5 min) - failure modes, monitoring
5. **Trade-offs** (5 min) - consistency, latency, cost

### Behavioral: STAR Stories to Prepare
Prepare 5-7 stories covering:
- [ ] Technical challenge overcome
- [ ] Disagreement with peer/manager resolved
- [ ] Project delivered under pressure
- [ ] Mistake made and learned from
- [ ] Mentored/upskilled someone
- [ ] Improved process/tooling
- [ ] Said "no" / pushed back on scope

## Salary Negotiation

### Research Phase
| Source | Use For |
|---------|---------|
| Levels.fyi | Big tech benchmarks |
| Glassdoor/Payscale | Company-specific ranges |
| Blind/Reddit | Unverified but directional |
| H1B Data | Verified salary floors |
| Recruiters | Current market pulse |

### Negotiation Script
```
"Thank you for the offer! I'm excited about the role and the team. 

Based on my research and [X years experience / specific expertise], I was targeting a base salary in the range of $X-$Y. 

Is there flexibility on the base? I'm also open to discussing equity/sign-on to bridge any gap."
```

### Negotiable Items (Priority Order)
1. Base salary
2. Signing bonus (easiest for them)
2. Equity/RSUs (especially pre-IPO)
3. Annual bonus target %
4. Vacation/PTO
5. Remote/hybrid flexibility
6. Education budget
7. Title (if close to next level)

## Offer Evaluation Framework

### Compare Offers Holistically
| Factor | Weight (1-5) | Offer A | Offer B | Offer C |
|--------|--------------|---------|---------|---------|
| Base Salary | 5 | | | |
| Total Comp (1yr) | 5 | | | |
| Equity Upside | 4 | | | |
| Team/Manager | 5 | | | |
| Tech Stack/Growth | 4 | | | |
| Work-Life Balance | 4 | | | |
| Company Trajectory | 3 | | | |
| Location/Remote | 3 | | | |
| Benefits | 2 | | | |
| **Weighted Score** | | | | |

### Red Flags in Offers
- Below market with "we pay in experience"
- No equity refresh policy
- Uncapped on-call without compensation
- Non-compete > 12 months or broad geography
- "We'll discuss promotion in 6 months" (get in writing)
- Pressure to decide in <48 hours
